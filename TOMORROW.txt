   - Add predicted_tier as a filter (T1_Entry / T2_Junior / T3_Senior / T4_Management)
   - Display as friendly labels: Entry Level / Junior / Senior / Management
   - Can be pre-filled based on resume's inferred experience level

C. API: expose role_cluster and predicted_tier in job responses
   - Current job response objects probably don't include these fields
   - Add them to the relevant GET /jobs and matching endpoints
   - Check src/mcf/api/server.py for the job serialization

Priority 2 — Multi-label role matching (more complex, high value)

D. Multi-label role tagging
   - At threshold 0.85 cosine similarity, 45% of jobs belong to 2+ role clusters
   - Implementation: at classification time, compute similarity to all 35 centroids
     and store all clusters above threshold (not just the nearest one)
   - Requires a separate column (e.g. role_clusters_json TEXT array) or junction table
   - Enables: resume scoring high on Data Science also surfaces Data Engineering jobs
   - Key co-occurring pairs: PM<->SWE, PM<->BD, SWE<->Data, QA<->UX

E. Use tier in matching score
   - Currently matching is purely cosine similarity + recency decay
   - Boost (or filter) match scores when candidate's inferred tier matches job's predicted_tier
   - Candidate tier comes from the resume embedding run through lr_tier_classifier
   - See src/mcf/api/services/matching_service.py

Priority 3 — Maintenance

F. Re-train classifiers when job pool changes significantly
   - Models are trained on Apr 2026 snapshot (69k jobs)
   - Scripts to retrain are already written (run_clustering_v2.py, run_role_clustering.py)
   - Suggested trigger: re-train quarterly or when job pool grows >20%
   - After retraining, replace src/mcf/models/*.pkl and run backfill again

G. Consider merging similar clusters if UI feedback shows confusion
   - C13 + C28 (both Kitchen/Culinary — same salary, different seniority)
   - C19 + C30 (both Software — developer vs senior/embedded split)
   - These are fine as-is for now but worth watching


KEY NUMBERS TO REMEMBER
=========================

Tier classifier (LogisticRegression):
  - Balanced accuracy: 0.875 (vs 0.698 for KNN)
  - Model size: 25 kB
  - Predict latency: 0.07 ms per 32-job batch

Role classifier (K-Means k=35):
  - Silhouette: 0.0955 (5-seed mean)
  - Predict latency: ~0.1 ms per batch (essentially free)

Multi-label threshold: 0.85
  - 45% of jobs match 2+ role clusters at this threshold
  - Avg 1.89 labels/job

Embedding model: BAAI/bge-base-en-v1.5 (768-dim)
  - Seniority + years experience baked into embedding text
  - Embedding text format: "Job Title: X\nRequired Skills: ...\nSeniority: Y, N+ years experience"
